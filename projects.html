Car Rental Management System

Streamlining Operations for a Digital Garage

View Source Code

Tech Stack

Java Java Swing MySQL NetBeans IDE

The Challenge

Small to medium-sized car rental companies often rely on manual record-keeping with spreadsheets or paper-based systems. This leads to inefficiencies, difficulty tracking vehicle availability, higher risk of human error in billing, and a poor overview of business performance.

The Solution

I designed and developed a comprehensive desktop application to serve as a centralized management hub. The system digitizes the entire rental lifecycle, from adding new vehicles to generating end-of-day financial reports, ensuring data accuracy and saving administrative time.

Key Features

  • Vehicle & Customer Database: Secure management of vehicle and customer records with full CRUD functionality.
  • Rental Transaction Processing: Automated handling of rentals and returns with integrated pricing calculations.
  • Real-time Availability Tracking: Automatic status updates (Available, Rented, Maintenance) to prevent double-booking.
  • Reporting Module: Generation of financial and operational reports for business intelligence.
  • User Authentication: Secure login system to protect sensitive business data.

Technical Implementation

The application follows a Model-View-Controller (MVC) architecture with a Java Swing GUI frontend and a MySQL database backend. I implemented custom event handlers for the UI components and used JDBC for database connectivity, ensuring robust data persistence and transaction management.

Tech Stack

Java Java Swing MySQL Figma

The Challenge

Traditional phone orders for pizza can be prone to errors during peak hours, lack a visual menu, and don't provide an easy way for customers to customize their orders or for owners to track sales trends.

The Solution

I created a user-friendly desktop application that simulates a modern pizza ordering experience. It allows customers to build their perfect pizza with a visual list of toppings, manages order fulfillment for the kitchen staff, and provides administrators with tools to update the menu and view sales data.

Key Features

  • Interactive Menu Interface: Visual selection of pizza sizes, crust types, and toppings with real-time price updates.
  • Shopping Cart System: Ability to add multiple items, review orders, and modify quantities before checkout.
  • Order Management Dashboard: Kitchen display system showing order status and preparation progress.
  • Admin Control Panel: Secure interface for menu management, price adjustments, and sales reporting.
  • Order History & Analytics: Tracking of sales data and customer order patterns for business insights.

Technical Implementation

The system features a responsive Java Swing interface with custom rendering of food items. I implemented a dynamic pricing engine that calculates costs based on user selections in real-time. The database design normalized menu items, orders, and customer data to ensure efficient querying and data integrity.

Pine City Zoo

Bringing the Zoo Experience Online

View Source Code

Tech Stack

HTML5 CSS3 JavaScript Figma

The Challenge

Potential visitors often want to preview a zoo's attractions before planning a visit. A static, informational website fails to capture the excitement and educational value of the animals, missing an opportunity to engage users and drive ticket sales.

The Solution

I designed and developed a dynamic, interactive virtual tour website for a fictional zoo. The site allows users to explore different animal exhibits, learn detailed facts in an engaging way, and easily find practical visit information, creating an online experience that mirrors the wonder of an actual visit.

Key Features

  • Interactive Habitat Sections: Themed areas (African Savannah, Amazon Rainforest) with curated animal displays.
  • Dynamic Animal Profiles: Clickable cards that reveal detailed information about species, habitat, and conservation status.
  • Responsive Design: Fully adaptive layout that works seamlessly on desktop, tablet, and mobile devices.
  • Visual Storytelling: High-quality imagery and intuitive navigation that creates an immersive experience.
  • Visit Planning Tools: Integrated information about tickets, hours, and directions to encourage actual visits.

Technical Implementation

Built with semantic HTML5 for accessibility, CSS3 with Flexbox/Grid for modern layouts, and vanilla JavaScript for interactive elements. I followed a mobile-first approach and used CSS transitions and transformations to create smooth animations. The design was first prototyped in Figma to ensure visual consistency and user experience quality.
